# 17.5 Veri Sınıflandırma ve İşleme Envanteri

## Önerilen Teknik Sözlük

Bu sözlük banka politikasıyla onaylanana kadar `Proposed` durumundadır:

| Kod | Teknik davranış |
| --- | --- |
| PUBLIC | Genel erişim politikası banka kararıyla belirlenir |
| INTERNAL | Banka içi yetkili kullanım |
| CONFIDENTIAL | Sınırlı rol ve kapsam |
| PERSONAL_DATA | Maskeleme ve KVKK envanter ilişkisi zorunlu |
| SPECIAL_CATEGORY_PERSONAL_DATA | Ek tedbir ve açık kapsam zorunlu |
| CUSTOMER_SECRET | Ham değer varsayılan olarak uygulama dışına çıkamaz |
| BANK_SECRET | Ham değer varsayılan olarak uygulama dışına çıkamaz |
| HIGHLY_RESTRICTED | Açık onay olmadan örnekleme, görüntüleme ve dışa aktarma yok |

## Mevcut Modele Eklenmesi Önerilen Alanlar

| Alan | Açıklama |
| --- | --- |
| classification_code | Onaylı sınıf kodu |
| classification_source | Manuel, katalog, sınıflandırıcı veya miras |
| processing_purpose | İşleme amacı |
| legal_basis | Banka hukuk/uyum tarafından onaylı referans |
| data_owner_id | Veri sahibi |
| data_steward_id | Veri sorumlusu/steward |
| retention_policy_id | Saklama politikası |
| masking_policy_id | Maskeleme politikası |
| export_policy_id | Dışa aktarma politikası |
| recipient_groups | Alıcı grubu |
| cross_border_transfer_flag | Yurt dışı aktarım etkisi |
| critical_data_element | Kritik veri öğesi işareti |
| regulatory_reporting_usage | Düzenleyici/risk raporlama kullanımı |

## Varsayılan Davranış

- Sınıf yoksa ham örnek gösterme.
- Profil sonucunda ham değer saklama.
- Teknik hata ve issue açıklamasında ham kayıt kullanma.
- ServiceNow'a uygulama içi yetkili bağlantı gönder; ham kayıt gönderme.
- Audit olayında sadece değişiklik özeti ve redaksiyon bilgisi tut.
