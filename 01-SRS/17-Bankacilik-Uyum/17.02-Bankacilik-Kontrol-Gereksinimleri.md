# 17.2 Bankacılık Kontrol Gereksinimleri

## Kimlik ve Yetki

| ID | Gereksinim |
| --- | --- |
| **BFR-IAM-001** | Sistem, her kullanıcı ve servis işlemini güvenilir kimlik adaptörü tarafından üretilen değişmez `ActorContext` ile ilişkilendirmelidir. |
| **BFR-IAM-002** | Sistem, çağıranın serbest metin aktör kimliği, rolü veya veri kapsamını yetki kanıtı olarak kabul etmemelidir. |
| **BFR-IAM-003** | Sistem, LDAP grup bilgisini banka tarafından onaylı sürümlü grup-rol-scope eşleme politikasıyla yerel yetkiye dönüştürmelidir. |
| **BFR-IAM-004** | Sistem, kimlik veya yetki adaptörü erişilemediğinde varsayılan olarak erişimi reddetmelidir. |
| **BFR-IAM-005** | Sistem, ayrıcalıklı ve break-glass oturumlarını normal kullanıcı oturumlarından ayırmalı, süreli ve gerekçeli olarak audit etmelidir. |
| **BFR-IAM-006** | Sistem, servis hesaplarını kişisel kullanıcılardan ayırmalı ve servis hesabı kapsamını çağrı amacıyla sınırlandırmalıdır. |

## Görevler Ayrılığı

| ID | Gereksinim |
| --- | --- |
| **BFR-SOD-001** | Sistem, kritik değişiklikleri maker-checker iş akışına alabilmelidir. |
| **BFR-SOD-002** | Sistem, değişikliği hazırlayan aktörün aynı değişikliği onaylamasını engellemelidir. |
| **BFR-SOD-003** | Sistem, kural aktivasyonu, skor konfigürasyonu, veri kaynağı aktivasyonu, rol değişikliği ve hassas dışa aktarma için ayrı onay politikaları desteklemelidir. |
| **BFR-SOD-004** | Sistem, onay, ret, süre aşımı ve geri çekme olaylarını değişiklik sürümüyle birlikte audit etmelidir. |

## Veri Koruma

| ID | Gereksinim |
| --- | --- |
| **BFR-DATA-001** | Sistem, veri alanlarını banka tarafından onaylı sınıflandırma sözlüğüyle etiketlemelidir. |
| **BFR-DATA-002** | Sistem, sınıfı belirlenmemiş alanlarda ham örnek göstermemeli ve deny-by-default maskeleme uygulamalıdır. |
| **BFR-DATA-003** | Sistem, log, audit, bildirim, issue ve ServiceNow payloadlarında ham kişisel veri, müşteri sırrı veya banka sırrı taşımamalıdır. |
| **BFR-DATA-004** | Sistem, kişisel veri işleme amacı, hukuki sebep, veri sahibi, saklama politikası ve aktarım etkisini metadata ile ilişkilendirebilmelidir. |

## Audit ve Kanıt

| ID | Gereksinim |
| --- | --- |
| **BFR-AUD-001** | Sistem, tüm modüller için ortak ve sürümlü bir audit olay zarfı kullanmalıdır. |
| **BFR-AUD-002** | Audit olayı aktör/servis kimliği, zaman, correlation ID, işlem, nesne, sonuç, önceki/sonraki değer özeti ve redaksiyon bilgisini taşımalıdır. |
| **BFR-AUD-003** | Sistem, audit kayıtlarının sıralama ve bütünlük doğrulamasını desteklemelidir. |
| **BFR-AUD-004** | Sistem, audit yazma hatasında kritik işlem sınıfına göre fail-closed veya dayanıklı tampon davranışını açık politikayla uygulamalıdır. |
| **BFR-AUD-005** | Sistem, audit görüntüleme ve dışa aktarma işlemlerini de audit etmelidir. |

## Dışa Aktarma ve Entegrasyon

| ID | Gereksinim |
| --- | --- |
| **BFR-EXP-001** | Sistem, hassas rapor dışa aktarmayı rol, veri kapsamı, amaç/gerekçe ve banka onay politikasına bağlamalıdır. |
| **BFR-EXP-002** | Sistem, dışa aktarma sonucunda oluşan dosya için sınıflandırma, üretim zamanı, filtreler, aktör ve bütünlük özeti saklamalıdır. |
| **BFR-EXP-003** | Sistem, yetkisiz veri alanlarını dışa aktarma dosyasına eklememelidir. |
| **BFR-EXT-001** | ServiceNow adaptörü yalnız onaylı alan whitelist'ini göndermelidir. |
| **BFR-EXT-002** | ServiceNow payloadı kural/execution/issue kimliği, özet sayaç ve yetkili uygulama bağlantısı dışında ham kayıt içermemelidir. |
| **BFR-EXT-003** | Harici entegrasyonlar timeout, idempotency, retry ve veri aktarım bölgesi kararlarını desteklemelidir. |

## Yaşam Döngüsü ve Olay Müdahalesi

| ID | Gereksinim |
| --- | --- |
| **BFR-LCM-001** | Sistem, kayıt türü bazlı saklama ve imha politikası uygulamalıdır. |
| **BFR-LCM-002** | Sistem, legal hold altındaki kayıtları otomatik imhadan hariç tutmalıdır. |
| **BFR-LCM-003** | Sistem, imha işlemini kayıt türü, yöntem, kapsam, aktör ve sonuçla audit etmelidir. |
| **BFR-LCM-004** | Sistem, arşivlenen denetim ve skor kayıtlarını yetkili geri çağırma süreciyle erişilebilir kılmalıdır. |
| **BFR-IR-001** | Sistem, kişisel veri ihlali şüphesini güvenlik olayından ayrı iş akışında izleyebilmelidir. |
| **BFR-IR-002** | Sistem, ihlalin öğrenilme zamanı, etkilenen kapsam, alınan önlem ve karar zaman çizelgesini korumalıdır. |
| **BFR-IR-003** | Sistem, 72 saatlik değerlendirme zamanını görünür kılmalı; Kurula otomatik bildirim göndermemelidir. |
| **BFR-IR-004** | Sistem, veri işleyen kaynaklı ihlalde veri sorumlusuna gecikmeksizin bildirim kanıtını saklayabilmelidir. |

| **BFR-SDLC-004** | Yazılım bileşen listesi/SBOM sürümle ilişkilendirilmelidir. |
