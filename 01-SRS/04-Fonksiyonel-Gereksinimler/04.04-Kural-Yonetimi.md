# Veri Kalitesi Kural Yönetimi

### FR-023 — Hazır kural şablonları

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-023** |
| Gereksinim adı | Hazır kural şablonları |
| Açıklama | Sistem tamlık, geçerlilik, kesin/olası benzersizlik, güncellik/zamanlılık, referans bütünlüğü, kaynak içi/kaynaklar arası tutarlılık ve mutabakat için şablon sunmalıdır. Teknik biçim geçerliliği ile iş anlamı geçerliliği ayrı kurallar olarak modellenebilmelidir. (`DQ-SCR-007`–`DQ-SCR-012`) |
| Aktör | Veri Kalitesi Uzmanı |
| Öncelik | Must |
| Girdi | Şablon türü ve parametreler |
| İşlem | Parametre doğrulama ve kural üretme |
| Çıktı | QualityRule taslağı |
| Ön koşullar | Yetkili kullanıcı; metadata mevcut |
| Son koşullar | Kural taslak olarak kaydedilir |
| İş kuralları | RULE-001, RULE-005 |
| Hata durumları | Eksik parametre; tip uyumsuzluğu |
| Kabul kriterleri | Her zorunlu şablon için olumlu ve olumsuz test verisiyle doğru SQL/işlem planı üretmelidir. |
| İlgili kullanım senaryosu | UC-005 |
| Bağımlılıklar | FR-015 |
