# Audit Log ve Denetim

### FR-077 — Kritik işlem audit kaydı

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-077** |
| Gereksinim adı | Kritik işlem audit kaydı |
| Açıklama | Sistem giriş, kaynak/kural/yetki/eşik/sorun değişikliği, rapor indirme ve başarısız erişimleri audit loga yazmalıdır. |
| Aktör | Sistem |
| Öncelik | Must |
| Girdi | İşlem olayı |
| İşlem | Normalize etme, hash/bütünlük bilgisi ekleme |
| Çıktı | AuditLog |
| Ön koşullar | Kritik işlem gerçekleşmiş |
| Son koşullar | Kayıt sorgulanabilir |
| İş kuralları | RULE-007, RULE-009 |
| Hata durumları | Audit deposu ulaşılamıyor |
| Kabul kriterleri | Kritik yazma işlemi audit kaydı oluşturulamıyorsa fail-closed veya güvenlik onaylı kuyruk politikası uygulanmalıdır. |
| İlgili kullanım senaryosu | UC-016 |
| Bağımlılıklar | Audit altyapısı |

### FR-078 — Audit filtreleme ve dışa aktarma

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-078** |
| Gereksinim adı | Audit filtreleme ve dışa aktarma |
| Açıklama | Sistem audit kayıtlarını kullanıcı, işlem, nesne, tarih, sonuç ve istemci bilgisine göre filtrelemeli ve yetkili dışa aktarma sağlamalıdır. |
| Aktör | Denetçi; Sistem Yöneticisi |
| Öncelik | Must |
| Girdi | Filtre ve format |
| İşlem | Sayfalama, maskeleme |
| Çıktı | Audit listesi/dosyası |
| Ön koşullar | Audit erişim yetkisi |
| Son koşullar | Dışa aktarma da auditlenir |
| İş kuralları | RULE-009 |
| Hata durumları | Aşırı geniş sorgu; yetkisiz alan |
| Kabul kriterleri | Beş yıllık aralık sorguları asenkron rapor olarak üretilebilmelidir. |
| İlgili kullanım senaryosu | UC-016 |
| Bağımlılıklar | FR-077 |

### FR-079 — Audit bütünlüğü

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-079** |
| Gereksinim adı | Audit bütünlüğü |
| Açıklama | Sistem audit kayıtlarının sonradan değiştirilmesini engellemeli ve bütünlük doğrulaması sağlamalıdır. |
| Aktör | Sistem Yöneticisi; Denetçi |
| Öncelik | Must |
| Girdi | Audit kayıtları |
| İşlem | Append-only saklama ve bütünlük kontrolü |
| Çıktı | Doğrulama sonucu |
| Ön koşullar | Audit altyapısı etkin |
| Son koşullar | Bozulma alarmı oluşabilir |
| İş kuralları | RULE-007 |
| Hata durumları | Depo bozulması; saat kayması |
| Kabul kriterleri | Yetkisiz UPDATE/DELETE denemeleri engellenmeli; periyodik bütünlük kontrolü raporlanmalıdır. |
| İlgili kullanım senaryosu | UC-016 |
| Bağımlılıklar | NFR-SEC |
