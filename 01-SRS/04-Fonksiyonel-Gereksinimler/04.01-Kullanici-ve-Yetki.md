# Kullanıcı ve Yetki Yönetimi

### FR-001 — Kurumsal SSO ile kullanıcı girişi

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-001** |
| Gereksinim adı | Kurumsal SSO ile kullanıcı girişi |
| Açıklama | Sistem, LDAP destekli kurumsal IdP/SSO üzerinden OIDC veya SAML kimlik beyanını doğrulamalı; tüm kullanıcılar için kurumsal IdP MFA kanıtı mevcutsa oturum oluşturmalıdır. Uygulama LDAP şemasına doğrudan bağımlı olmamalıdır. |
| Aktör | Tüm insan kullanıcılar |
| Öncelik | Must |
| Girdi | Kurumsal IdP kimlik beyanı, MFA doğrulama bilgisi ve grup üyelikleri |
| İşlem | IdP beyanı doğrulama, MFA kontrolü ve kullanıcı/grup-rol eşleme |
| Çıktı | Aktif oturum veya doğrulama hatası |
| Ön koşullar | Kurumsal IdP erişilebilir; kullanıcı aktif; MFA tamamlanmış |
| Son koşullar | Oturum ve giriş audit kaydı oluşur |
| İş kuralları | RULE-001, RULE-009 |
| Hata durumları | IdP ulaşılamıyor; geçersiz beyan; MFA eksik/başarısız; pasif kullanıcı; rol çatışması |
| Kabul kriterleri | Geçerli SSO beyanı ve MFA kanıtı olan kullanıcı 5 saniye içinde giriş yapabilmeli; MFA eksik/başarısızsa veya IdP erişilemiyorsa yerel girişe düşmeden oturum reddedilmelidir. |
| İlgili kullanım senaryosu | UC-001 |
| Bağımlılıklar | Kurumsal IdP/SSO, LDAP destekli kurumsal dizin, audit log |

### FR-005 — Oturum yönetimi

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-005** |
| Gereksinim adı | Oturum yönetimi |
| Açıklama | Sistem BFF mimarisinde sunucu taraflı opak normal kullanıcı oturumu; 1 saat boşta kalma, 10 saat mutlak süre, tek aktif oturum, merkezi iptal ve güvenli çıkış kurallarını uygulamalıdır. Tarayıcı access veya refresh token'a erişmemelidir. |
| Aktör | Tüm kullanıcılar |
| Öncelik | Must |
| Girdi | `__Host-session` opak cookie, CSRF synchronizer token, son etkinlik zamanı |
| İşlem | Cookie, CSRF, süre, bütünlük, merkezi iptal ve eş zamanlı oturum doğrulaması |
| Çıktı | Geçerli oturum veya yeniden giriş talebi |
| Ön koşullar | Başarılı giriş |
| Son koşullar | Çıkışta/timeoutta oturum sırrı geçersizleşir ve silinir; veri-minimum güvenlik metadatası 90 gün saklanır |
| İş kuralları | RULE-009 |
| Hata durumları | Süresi geçmiş/değiştirilmiş cookie; eksik veya geçersiz CSRF kanıtı; merkezi depo/iptal hatası; ikinci oturumla iptal edilmiş credential |
| Kabul kriterleri | Boşta kalma süresi 60 dakika, mutlak süre 10 saat ve aktif oturum limiti 1 olmalıdır. Yeni başarılı giriş önceki oturumu iptal etmeli; arka plan isteği idle süresini, token yenileme mutlak süreyi uzatmamalıdır. Cookie `__Host-session`, `Secure`, `HttpOnly`, `SameSite=Lax`, `Path=/` ve domainsiz olmalı; girişte ve ayrıcalık değişikliğinde döndürülmelidir. State-changing istek synchronizer token custom header, Origin/Referer, Fetch Metadata ve CORS allowlist kontrollerini geçmelidir; `GET` durum değiştirmemelidir. Çıkış, süre aşımı, yeni giriş, kullanıcı pasifleştirme, kritik rol değişikliği, güvenlik olayı veya IdP iptali sonrası credential tekrar kullanılamamalıdır. |
| İlgili kullanım senaryosu | UC-001 |
| Bağımlılıklar | FR-001, NFR-SEC |
