# Kullanıcı ve Yetki Yönetimi

### FR-001 — LDAP ile kullanıcı girişi

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-001** |
| Gereksinim adı | LDAP ile kullanıcı girişi |
| Açıklama | Sistem kullanıcı kimlik bilgilerini LDAP üzerinden doğrulamalı ve başarılı doğrulamada kullanıcı oturumu oluşturmalıdır. |
| Aktör | Tüm insan kullanıcılar |
| Öncelik | Must |
| Girdi | Kullanıcı adı, parola veya kurumun LDAP kimlik bilgisi |
| İşlem | LDAP doğrulaması ve kullanıcı/grup eşleme |
| Çıktı | Aktif oturum veya doğrulama hatası |
| Ön koşullar | LDAP erişilebilir; kullanıcı aktif |
| Son koşullar | Oturum ve giriş audit kaydı oluşur |
| İş kuralları | RULE-001, RULE-009 |
| Hata durumları | LDAP ulaşılamıyor; hatalı kimlik bilgisi; pasif kullanıcı |
| Kabul kriterleri | Geçerli kullanıcı 5 saniye içinde giriş yapabilmeli; hatalı girişte oturum oluşmamalıdır. |
| İlgili kullanım senaryosu | UC-001 |
| Bağımlılıklar | LDAP, audit log |

### FR-002 — Rol bazlı erişim kontrolü

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-002** |
| Gereksinim adı | Rol bazlı erişim kontrolü |
| Açıklama | Sistem her ekran, API ve veri nesnesi isteğinde RBAC yetkisini doğrulamalı ve yetkisiz işlemi engellemelidir. |
| Aktör | Sistem Yöneticisi; tüm kullanıcılar |
| Öncelik | Must |
| Girdi | Kullanıcı, rol, izin, kaynak kapsamı |
| İşlem | Yetki matrisi değerlendirmesi |
| Çıktı | İzin veya 403 erişim reddi |
| Ön koşullar | Kullanıcı doğrulanmış |
| Son koşullar | Karar audit/güvenlik kaydına yazılır |
| İş kuralları | RULE-001, RULE-009 |
| Hata durumları | Rol eksik; kapsam dışı nesne; izin servisi hatası |
| Kabul kriterleri | Yetkisiz 100 test isteğinin 100'ü engellenmeli; yetkili istekler doğru sonuç vermelidir. |
| İlgili kullanım senaryosu | UC-001, UC-016 |
| Bağımlılıklar | FR-001, Permission verisi |

| Alan | Tanım |

| Gereksinim ID | **FR-003** |
| Gereksinim adı | Kullanıcı ve rol eşleme |
