# Raporlama

Olay zaman çizelgesi ve değişmez kanıt paketi ikinci fazda `FR-111` ile
tanımlanır. Teknik kanıt paketi, denetim raporu veya banka uyum onayıyla eş anlamlı
değildir; erişim, sınıflandırma, saklama ve dışa aktarma politikaları ayrıca uygulanır.

### FR-072 — Özet ve detay raporları

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-072** |
| Gereksinim adı | Özet ve detay raporları |
| Açıklama | Sistem ham/nihai kalite skoru, ölçüm yeterliliği, teknik çalışma, kritik kural, kullanım kararı, kapsam/güven, geçerlilik, kural/boyut kırılımı ve tarihsel trendi ayrı gösteren özet ve detay raporları üretmelidir. |
| Aktör | Data Owner; Data Steward; Denetçi |
| Öncelik | Must |
| Girdi | Kapsam, tarih, filtre |
| İşlem | Yetkili veri sorgulama ve rapor düzenleme |
| Çıktı | Rapor görünümü |
| Ön koşullar | Sonuçlar mevcut |
| Son koşullar | Report kaydı oluşabilir |
| İş kuralları | RULE-009 |
| Hata durumları | Büyük veri; timeout |
| Kabul kriterleri | Rapor kullanılan filtreyi, oluşturulma zamanını, iş/snapshot dönemini, son başarılı çalışma ve geçerlilik zamanını, model/politika/referans/uygulama sürümlerini göstermeli; `TechnicalFailure`, `Stale` veya `LimitedCoverage` durumunu düşük kalite skoru olarak sunmamalıdır. |
| İlgili kullanım senaryosu | UC-015 |
| Bağımlılıklar | FR-054, FR-044 |

### FR-075 — PDF, Excel ve CSV dışa aktarma

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-075** |
| Gereksinim adı | PDF, Excel ve CSV dışa aktarma |
| Açıklama | Sistem yetkili kullanıcının raporu PDF, Excel ve CSV olarak dışa aktarmasını sağlamalıdır. |
| Aktör | Tüm yetkili kullanıcılar |
| Öncelik | Must |
| Girdi | Rapor ve format |
| İşlem | Harici sınıflandırmaya göre formatlama, maskeleme, raporlama kısıtı ve erişim kontrolü |
| Çıktı | Politika kontrollü çevrimiçi veya arşivlenmiş rapor dosyası ve ayrı metadata |
| Ön koşullar | Dışa aktarma yetkisi |
| Son koşullar | Rapor indirme audit kaydı oluşur |
| İş kuralları | RULE-009, RULE-010 |
| Hata durumları | Çok büyük çıktı; yetki; üretim hatası |
| Kabul kriterleri | Rapor dosyası, metadata, arşiv ve imha kaydı ayrı yönetilmeli; rapor dosyası `RET-30D-EXPORT`, metadata `RET-10Y-BANKING` ile saklanmalıdır. Dosya boyutu aktif sürümlü rapor politikasından alınmalı; politika yoksa üretim reddedilmelidir. Excel/CSV sınırı aşıldığında bölme veya güvenli hata açıklaması sunulmalıdır. |
| İlgili kullanım senaryosu | UC-015 |
| Bağımlılıklar | Dosya servisi, audit |
