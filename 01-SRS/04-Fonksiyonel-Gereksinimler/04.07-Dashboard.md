# Dashboard ve Görselleştirme

### FR-054 — Genel dashboard

| Alan | Tanım |
| --- | --- |
| Gereksinim ID | **FR-054** |
| Gereksinim adı | Genel dashboard |
| Açıklama | Sistem ham veri kalitesi skorunu, kural/ölçüm kapsamını, skor güvenilirliğini, dataset kritiklik profilini/veri riskini ve teknik sağlık durumunu ayrı göstergeler olarak sunmalı; dataset → boyut → veri öğesi → kural kırılımını ve provizyonel sonucu açıklamalıdır. Skor personel performans puanı olarak etiketlenmemelidir. (`DQ-SCR-001`, `DQ-SCR-002`, `DQ-SCR-018`–`DQ-SCR-021`, `DQ-SCR-026`) |
| Aktör | Tüm yetkili kullanıcılar |
| Öncelik | Must |
| Girdi | Kullanıcı kapsamı, tarih aralığı |
| İşlem | Yetkili verileri toplulaştırma |
| Çıktı | KPI kartları ve grafikler |
| Ön koşullar | Skorlar mevcut |
| Son koşullar | Veri değişmez |
| İş kuralları | RULE-009 |
| Hata durumları | Veri yok; sorgu timeout |
| Kabul kriterleri | Varsayılan son 30 gün görünümü önerilen hedefte 3 saniye içinde açılmalı; provizyonel sonuç “Geçici” veya “Provizyonel”, kapsama ve eksik koşullarla gösterilmelidir. “Yüksek kalite, düşük güven” ve eski son başarılı skor açıkça ayırt edilmelidir. |
| İlgili kullanım senaryosu | UC-010 |
| Bağımlılıklar | FR-050, FR-002 |
